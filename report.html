Hotel Booking Website Report

Introduction

This project is a frontend hotel booking website developed as a university assignment. The main goal was to design a clean and responsive interface using only XHTML, CSS, and JavaScript. The website supports key tasks such as exploring categories, viewing room details, and calculating reservation cost by number of nights. This project improved my practical understanding of page structure, styling, and client-side validation.

Body

1) Development Process

I built the project in three connected pages. First, index.html works as the homepage with the website title, logo, vision and mission, developer profile (Asim Khursheed), navigation links, and a contact section with email, phone, and a simple form. Second, categories.html presents five required categories: Luxury Hotels, Business Hotels, Family Hotels, Budget Hotels, and Beach Resorts. I used modern cards with hover effects, and each card links to rooms.html. Third, rooms.html displays three room options with real photos, descriptions, features, and per-night prices. In script.js, I added the reservation formula: total = price x nights. The script validates input and rejects empty, zero, or negative values with user-friendly output. Because this is a static frontend project, there is no backend storage or live booking submission.

3) Future Improvements

  • Online payment integration for secure checkout.
  • User login/signup for personalized booking history.
  • Real-time booking status for live room availability.

Conclusion

In conclusion, this project successfully demonstrates a complete static hotel booking website with category browsing, room presentation, and dynamic cost calculation. The assignment improved my skills in frontend layout planning, responsive design, and JavaScript validation. I also learned the difference between static prototypes and full production booking systems, which motivates future backend integration.
