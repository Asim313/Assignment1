# Hotel Booking Website Report

## Introduction
This project is a frontend Hotel Booking Website developed as a university assignment. The objective was to build a responsive interface using only XHTML, CSS, and JavaScript. The website allows users to navigate hotel categories, check room information, and calculate booking cost according to the number of nights. Through this project, I improved my understanding of page structure, visual design, and client-side validation.

## Body
### 1) Development Process
The website was developed in three connected pages. The homepage includes the title, logo, vision, mission, developer profile (Asim Khursheed), navigation menu, and a contact section with email, phone number, and a simple form. The second page presents the required categories: Luxury Hotels, Business Hotels, Family Hotels, Budget Hotels, and Beach Resorts. These categories are displayed as cards with hover effects, and each card redirects users to the rooms page. The third page displays three rooms with real images, descriptions, features, and price per night. In JavaScript, I implemented the formula **total = price x nights** to calculate cost dynamically. Input validation rejects empty, zero, or negative values and shows feedback.

Because this is a static frontend project, there is no backend storage.

### 2) Competitor Analysis

| Platform | Advantages (2) | Disadvantages (2) | Comparison with My Website |
|---|---|---|---|
| Booking.com | Large hotel database; trusted reviews and filters | Interface can feel crowded; paid promotions may affect ranking | My website is simpler and easier to use, but it has fewer listings and no review system |
| Airbnb | Unique stay options; strong host-guest communication | Property quality can vary; service and cleaning fees may be high | My website is hotel-focused and straightforward, but it has no account system or host interaction tools |

### 3) Future Improvements
To make this project closer to real-world systems, I plan to add:
1. Online payment integration for secure and smooth checkout.
2. User login and signup for account management and booking history.
3. Real-time booking status for live room availability.

## Conclusion
In conclusion, this project demonstrates a static hotel booking website with category browsing, room display, and dynamic cost calculation. This assignment improved my skills in layout planning, responsive design, and JavaScript input validation. It also helped me understand the difference between a frontend prototype and a functional booking platform with backend services.
